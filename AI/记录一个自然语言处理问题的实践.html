# 记录一个自然语言处理问题的实践

# 问题描述

本实验的问题是将商品名称分类到对应标签。商品名称较短，是为短文本。标签数量太多，是难分问题。数据集太脏，错误比较多。

# 文本分词

首先需要进行文本的切分，对比了结巴，清华，北大的分词工具，分词时间太长。最终选择了 jieba分词工具。

# 生成特征向量

  • 一开始的打算，分词 + one-hot编码 + 神经网络。但经过计算 one-hot 编码的占用内存空间太大了。总词数 * 句子树的特征矩阵。

  • 后来尝试 label-encoding 编码（最长句子的词数 * 句子数）。 但是效果不好，原因估计是并不能很好的区分词之间的相似度。

  • word-Embeding。 这样的实践做多了，确实见识就多了。又认识一种新的编码方案。 所谓 word-Embeding 就是将词映射到一个高维空间。如 CBOW ，TF-IDF等算法都可以，结合实际情况选择合适的方案。而本实验使用的是 CBOW。 它就是一个浅层神经网络，通过将一个句子的词 one-hot作为输入，将输出作为句向量。而这个向量的长度可以被定义，长度越长词语之间的相关性越弱。之所以使用这个是因为这个是 Facebook 开源的 FastText 模型的内置算法。这样的高维空间可以很好的表示句子之间的相关性。

# 选择分类器（模型）
  • CNN神经网络。先跑通一次吧，从神经网络开始。时间太长，效果不好。
  • FastText。 在网上搜集资料，了解了FastText模型。写了一个Demo之后，发现这个模型训练时间短，效果还不错。组内大佬把这个模型的输入输出接口改了一下，使它符合我们的输入输出规范，还让它可以多线程预测，这使得预测速度在我们实验的硬件环境上加快了14.5倍！
  • 层次分类模型（树型）。老师帮我们找到一篇指导方向的论文，通过这篇文章，我将目光放在了层次分类上。之前说到标签数量太多，分类种类太多自然会影响结果。一般来说，分类种类多，所需要的特征向量也越长。打算通过增加层次进行分层训练，期望达到好的结果，实际上标签确实是有层次的(ep. /游戏/腾讯游戏/王者荣耀)。于是我在GitHub上找到了一个分层分类工具，但它只支持 sklearn 的模型。因此我也改了改让它可以支持我们修改后的FastText。但是效果与之前类似，并无明显提升。我尝试修改分层模型的预测方式，将贪心模式改为分支限界算法，效果仅有0.1%左右的提升。经过进行每个节点的正确率分析。总结原因是数据太脏，在某些节点上的表现不好。事实上，所谓的错误，在我们看了实际是正确的:)
