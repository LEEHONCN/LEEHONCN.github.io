# mac 配置 spark

# 安装 Spark下载地址

  1. 需要有 JDK 环境
  2. 下载完，解压即可。

# 配置 standalone 模式

  1. 进入 spark 目录
    cd /Users/lh_os/Desktop/spark-2.4.0-bin-hadop2.7
  2. 进入 spark 交互环境
    ./bin/pyspark

# 配置 singlenode cluster模式 （集群）

  1. 修改conf/slaves 文件，增加localhost(可能本来就有)

    在 /conf/spark-env.sh 中添加

    export PYSPARK_PYTHON=path
export PYSPARK_DRIVER_PYTHON=path

    其中 path 为你的python路径。要注意该路径的python版本要和spark匹配，否则最后一步会报 Exception: Python in worker has different version 2.7 than that in driver 3.6 异常。

  2. 启动 spark 主服务

    ./sbin/start-master.sh

    查看 http://localhost:8080。此时该页面中下边三项应该都是0

  3. 增加 worker

    ./sbin/start-slaves.sh spark://127.0.0.1:7077

    • 过程中可能配置ssh,输入ssh密码。

    • 查看 http://localhost:8080。此时该页面workers后边变成1

    • ps:如果遇到localhost: ssh: connect to host localhost port 22: Connection refused错误。 在系统偏好设置 -》 共享 -》勾选远程登录

    • 如果没有 workers (可能因为中间切换了网络)，则

      ./sbin/stop-all.sh
./sbin/start-all.sh

      先关闭，再重新开启(执行 2 3的指令)。

    • 如果遇到 master 和 worker 的地址不一样的话（由于中途切换了网络）。需要

      ./sbin/stop-all.sh
./sbin/start-all.sh

      重启一下。否则在最后一步，无法请求worker的地址，无法创建执行程序。

  4. 让worker 连接到 master 并进入spark交互环境

    ./bin/pyspark --master spark://127.0.0.1:7077
    • 在过程中可能出现 不能连接到master的错误 在网上找了半天，最后发现只需要将 http://localhost:8080 中最上方的黑体字 "Spark Master at spark://XXXX:XXXX" 的 “spark://XXXX:XXXX” 替换 spark://127.0.0.1:7077 即可

  5. 运行Spark例子（退出上一步的交互式环境）：

    ./bin/spark-submit  --master spark://127.0.0.1:7077 ./examples/src/main/python/pi.py 1000
    • 同样将 “spark://XXXX:XXXX” 替换 spark://127.0.0.1:7077
    • 查看 http://localhost:8080 中的信息，发现 Completed Applications 中出现了内容。运行示例成功。
    • Exception: Python in worker has different version 2.7 than that in driver 3.6。 添加
          export PYSPARK_PYTHON=XXXXXX
      正确版本的python 到环境变量。并重启
      ./sbin/stop-all.sh
./sbin/start-all.sh
