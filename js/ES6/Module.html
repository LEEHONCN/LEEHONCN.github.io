                      # Module

                      # ES6

                      ES6的模块自动采用严格模式。

                      export的三种写法

                      export 是实时绑定导出。

                      export var firstName = 'Michael';
// 或
export {firstName};
// 或
export {firstName as name};

                      import 的写法

                      不能修改绑定，存在引入提升 。Import 是单例模式

                      import * 不会引入默认导入。

                      // file a.js
export function a() {}
export function b() {}
// file b.js
export default function() {}

import {a, b} from './a.js'
import XXX from './b.js'
import './a.js';
import * as a from './a.js';

                      如果在一个模块输入后输出一个模块，那么可以这样写。

                      export { foo, bar } from 'my_module';

                      模块的继承

                      export * from 'circle';
export var e = 2.71828182846;
export default function(x) {
  return Math.exp(x);
}

                      # CommonJs

                      CommonJs 是 Node 环境的模块方案

                      // a.js
module.exports = {
    a: 1
}
// or
exports.a = 1

// b.js
var module = require('./a.js')
module.a // -> log 1

                      # 加载

                      浏览器加载：

                      <script src="path/to/myModule.js" defer></script>   // 最后加载，有序
<script src="path/to/myModule.js" async></script>		// 下载完加载，无序
<script type="module" src="./foo.js"></script>  	  // 加载 ES6  默认 defer

                      引入的模块有几点问题

                      • 代码是在模块作用域中运行，外部不可访问。
                      • 模块之中，也可以使用import export 绝对Url 或相对Url
                      • this 为 undefined

                      Node 加载：

                      Node默认使用 CommonJs 模块格式，若想使用ES6模块，需要采用.mjs 后缀文件名。Node 的 import 命令是异步加载的。

                      如果脚本文件省略了后缀名：

                      • 会依次尝试 ：./x.mjs ./x.js ./x.json ./x.node
                      • 如果都不存在会去加载 ./x/package.json的main字段指定的脚本
                      • 如果 main 字段不存在，依次尝试./x/index.mjs、./x/index.js、./x/index.json、./x/index.node。
                      • 如果还是不存在 则抛出错误。

                      ES6模块加载CommonJs模块

                      ES6模块中比CommonJs 少了几种顶层变量。 尽量不要使用这些顶层变量，否则可以使用 CommonJs 模块导出，然后再用ES6导入。ES6（import ）导入，会将 CommonJS的导出当做默认导出。

                      这种导入 CommonJS 的值拷贝依然生效

                      module.exports = {
  foo: 'hello',
  bar: 'world'
};

// 等同于
export default {
  foo: 'hello',
  bar: 'world'
};

// 写法一
import baz from './a';
// 写法二
import {default as baz} from './a';
// 写法三
import * as baz from './a';
// 其中 baz是对象。如果导出的是函数的话需要
bar.default();

                      CommonJs模块加载ES6模块

                      不使用 require 而要使用 import() 函数。ES6 模块的所有输出接口，会成为输入对象的属性。default也会变成default 属性。

                      # 循环加载

                      CommonJs加载机制

                      Node 加载完一个模块时，会将该模块生成一个对象并加载到内存中。若该模块完全加载完毕，则以后加载就会从内存中读取（值拷贝)。因为是值拷贝，所以对于基本类型的变量。获取的值是当时导入的值，而引用类型不存在这个问题（总是最新的值）。所以，当我们加载（尤其是循环加载）时，不应该使用 var foo = require('a').foo; 这会导致加载时，一个未完成状态的变量或函数被拷贝。 应该总是var foo = require('a');

                      CommonJs循环加载

                      • 执行要加载的模块a
                      • 如果a在未加载完毕的情况下，加载了其他模块b，则执行b。
                      • 若 b 中也加载了 a ，那么此时读取a 只能读取已经完成的部分。
                      • b 加载完成， 返回到 a模块继续加载。
                      • a 模块加载完毕，并更新 a模块在内存中的对象。

                      ES6加载机制

                      由于ES6模块加载是动态输出绑定，能保证在任何时候都能获取其他模块当前的最新值。这样需要保证，取值的时候能够取到值。

                      Import 会优先被静态分析，优先于模块的其他内容。因此下边输出的第一句不是 ”foo is running“。

                      同时 export 会动态绑定，因此值会被其他模块改变。 export 也有提升声明的作用，因为执行 bar模块的时候 foo 模块的foo变量还没有声明和赋值，却显示的是undefined。

                      // ./foo.js
console.log('foo is running');
import {bar} from './bar'
console.log('bar = %j', bar);
setTimeout(() => console.log('bar = %j after 500 ms', bar), 500);
export let foo = false;
console.log('foo is finished');

// ./bar.js
console.log('bar is running');
import {foo} from './foo';
console.log('foo = %j', foo)
export let bar = false;
setTimeout(() => bar = true, 500);
console.log('bar is finished');

node foo.js
// bar is running
// foo = undefined
// bar is finished
// foo is running
// bar = false
// foo is finished
// bar = true after 500 ms

                      # 转码

                      babel： ES6 转 ES6

                      ES6 module transpiler： 将 ES6模块转为CommonJs模块 或 AMD 模块

                      SystemJs：加载 ES6 模块、AMD 模块和 CommonJS 模块，将其转为 ES5 格式

                      对于 CommonJS 和 ES6 中的模块化的对比：

                      • 前者支持动态导入，也就是 require(${path}/xx.js)，后者是静态编译。目前不支持，但是已有提案（import()）

                      • 前者在导出时都是值拷贝，就算导出的值变了，导入的值也不会改变，所以如果想更新值，必须重新导入一次。但是后者采用实时绑定的方式，导入导出的值都指向同一个内存地址，所以导入值会跟随导出值变化。而且不同模块导入的是同一个实例。

                      • 前者是同步导入，因为用于服务端，文件都在本地，同步导入即使卡住主线程影响也不大。而后者是异步导入，因为用于浏览器，需要下载文件，如果也采用同步导入会对渲染有很大影响

                      • 后者会编译成 require/exports (CommonJs)来执行的

                      • CommonJs的模块中的 this 指向当前模块，ES6模块中的this是undefined

                      • 注意 CommonJs 和 ES6 的加载机制的不同点。主要是原因其实就是前面的一二两点（动态和静态导入，值绑定和实时绑定）
