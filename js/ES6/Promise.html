                      # Promise

                      # Promise概述

                      promise是为解决异步处理回调函数嵌套深度问题而产生的。。既然是解决回调金字塔问题，就不要把Promise也写成金字塔风格。

                      Promise是微任务（本轮事件循环结束执行）。

                      Promise 新建后就会立即执行。

                      Promise的构造函数中的resolve和reject不会终止程序。

                      Promise构造函数中的返回值只是为了终止程序，返回的表达式也可以执行。

                      有了Promise对象，就可以将异步操作以同步操作的流程表达出来，避免了层层嵌套的回调函数。此外，Promise对象提供统一的接口，使得控制异步操作更加容易。

                      两个特点

                      • 对象的状态不受外界影响。Promise对象代表一个异步操作，有三种状态：pending（进行中）、fulfilled（已成功）和rejected（已失败）。只有异步操作的结果，可以决定当前是哪一种状态，任何其他操作都无法改变这个状态。
                      • 一旦状态改变，就不会再变，任何时候都可以得到这个结果。Promise对象的状态改变，只有两种可能：从pending变为fulfilled和从pending变为rejected。

                      Promise缺点

                      • 无法取消Promise，一旦新建它就会立即执行，无法中途取消。
                      • 如果不设置回调函数，Promise内部抛出的错误，不会反应到外部。
                      • 当处于pending状态时，无法得知目前进展到哪一个阶段（刚刚开始还是即将完成）。

                      如果某些事件不断地反复发生，一般来说，使用 Stream 模式（什么是Stream模式）是比部署Promise更好的选择。

                      # 用法

                      • 基本用法
                      const promise = new Promise(function(resolve, reject) {
  // ... some code
  if (/* 异步操作成功 */){
    resolve(value);
  } else {
    reject(error);
  }
	//  调用resolve或reject并不会终结 Promise 的参数函数的执行
  //  是同步任务
  console.log('我在');
	return;
	console.log('我不在');
});

// 其中resolve的参数会传入then方法的success。
// 其中reject的参数会传入then方法的failure。
promise.then(function(value) {
  // success
}, function(error) {
  // failure
});
                      • p1的状态会传递给p2
                      const p1 = new Promise(function (resolve, reject) {
  // ...
});

const p2 = new Promise(function (resolve, reject) {
  // ...
  resolve(p1);
})

                      # Promise链式调用

                      • 若返回值为Promise，则该Promise的状态影响下一个then的执行。它们的状态相同，打印出来的字符串相同，但是不相等。
                      const p1 = new Promise(function (resolve, reject) {
    resolve(1);
});
const p2 = new Promise(function (resolve, reject) {
    resolve(2);
});

// 其中
const p3 = p1.then(function (value) {
    console.log(value);
    return p2;
});
p3.then(function (value) {
    console.log(p2);
    console.log(p3);
    console.log(p2 == p3);
    console.log(value);
});

// 等价于链式调用
p1.then(function (value) {
    console.log(value);
    return p2;
}).then(function (value) {
    console.log(p2);
    console.log(p3);
    console.log(p2 == p3);
    console.log(value);
});

// 1
// Promise { 2 }
// Promise { 2 }
// false
// 2

                      • 返回一个同步值（返回值为非Promise和Error）

                        传递给下一个then

                      • 当中间的then传入的不是函数的时候，会造成穿透。

                      • 若没有返回值，下一个then不会等待这个then结束才执行

                      • 不能返回自身

                      # 异常

                      • 若返回值为Error，则该错误不会被 catch捕获。会传给下一个then。

                        • 因为返回任意一个非 promise 的值都会被包裹成 promise 对象，即 return new Error('error!!!') 等价于 return Promise.resolve(new Error('error!!!'))。

                      • 若throw一个异常，则会被后边的catch捕获。

                        • 用回调的话该错误就会被吞噬掉，但是用promises我们可以轻松的在catch()方法里处理掉该错误。

                      # then、catch、finally

                      • Promise.prototype.then()
                        • then方法返回的是一个新的Promise实例（不是原来那个Promise实例）。因此可以采用链式写法
                        • then 方法内部有三种操作情况
                          • 返回同步值
                          • 返回Promise
                          • 抛出异常
                      • Promise.prototype.catch()
                        • catch方法返回的还是一个 Promise 对象
                        • Promise.prototype.catch方法是.then(null, rejection)或.then(undefined, rejection)的语法糖
                        • 可以捕获reject和程序抛出的异常，用于指定发生错误时的回调函数。
                        • Promise 在resolve语句后面，再抛出错误，不会被捕获，等于没有抛出。因为 Promise 的状态一旦改变，就永久保持该状态，不会再变了。
                        • Promise 对象的错误具有“冒泡”性质，会一直向后传递，直到被捕获为止。也就是说，错误总是会被下一个catch语句捕获。
                        • 不要在then方法里面定义 Reject 状态的回调函数（即then的第二个参数），总是使用catch方法。
                        • 尽量在每个Promise链式调用的最后使用catch方法
                      • Promise.prototype.finally()
                        • 不管promise最后的状态，在执行完then或catch指定的回调函数以后，都会执行finally方法指定的回调函数。
                        • finally方法里的操作应该是状态无关的

                      # resolve、reject

                      • Promise.resolve()
                        • 将现有对象转为 Promise 对象
                        • 参数情况
                          • 参数是一个 Promise 实例：直接返回。
                          • 参数是一个thenable对象：转为Promise对象，并执行then方法。
                          • 参数不是具有then方法的对象，或根本就不是对象：转为Promise对象，并将状态转为resolve。
                          • 不带有任何参数：直接返回resolve状态的Promise对象。
                        • 可以很好的利用后续的catch处理同步异常的问题
                      • Promise.reject()
                        • Promise.reject(reason)方法也会返回一个新的 Promise 实例，该实例的状态为rejected。

                      # all、race

                      • Promise.all()

                      • const p = Promise.all([p1, p2, p3]);
                        • Promise.all方法用于将多个 Promise 实例，包装成一个新的 Promise 实例。
                        • 参数列表中所有的Promise都已完成，p的状态才会变成完成态。若有一个被拒绝，它的状态会变成拒绝态。
                        • 如果参数列表中的Promise定义了catch方法，那么不会调用Promise.all().catch()方法。
                        • 你想在Promise中使用循环，那么你可能需要Promise.all

                      • Promise.race()

                        const p = Promise.race([p1, p2, p3]);
                        • Promise.race方法同样是将多个 Promise 实例，包装成一个新的 Promise 实例。
                        • 上面代码中，只要p1、p2、p3之中有一个实例率先改变状态，p的状态就跟着改变。那个率先改变的 Promise 实例的返回值，就传递给p的回调函数。

                      # try

                      • Promise.try()
                        • 让同步函数和异步函数有统一的API
                        • 解决被Promise包装的同步函数会处于本轮事件循环的末尾
                        • 可以更好地管理异常，有些API可能出现同步错误，也可能出现异步错误。try() 后面的**catch()**可以捕获这两种异常。

                      # then() 用法辨析

                      // 写法一   f2不接受f1的参数
f1().then(function(a){
  return f2();
}).then(function(a){ 
});

// 写法二   两个then并行
f1().then(function(a){
  f2();
}).then(function(a){
});

// 写法三   穿透，第一个then的参数不是函数
f1().then(f2()).then(function(a){});

// 写法四   f2接受f1的参数  
f1().then(f2).then(function(a){});

                      # 总结

                      Promise就是用来取代回调函数的API。

                      可以将API分为四类：

                      • then、catch、finally：为了取代回调函数
                      • Promise.resolve、Promise.reject ：用于简化某些使用情况
                      • Promise.all、Promise.race：为了满足某些使用需求
                      • Promisr.try ：为了解决同步异步API统一的问题

                      注意当then中返回值的不同，所呈现的差异。

                      需要知道是catch并没有什么特别之处，它仅仅是then的语法糖。所以then中返回值的差异对catch同样有效。
