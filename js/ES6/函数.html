                      # 函数

                      # 默认参数值

                      • arguments对象与命名参数分离(只当某个命名参数使用默认参数值时)，并且之后对命名参数的修改不会更新到arguments对象上（相当于ES5严格模式）。

                        // ES5 默认参数值
function log(x, y) {
  y = y || 'World';
  console.log(x, y);
}

                        function fn(a, b){
	"use strict";
  console.log(a === arguments[0]);
  console.log(b === arguments[1]);
  a = 1; 
  b = 2;
  console.log(a === arguments[0]);
  console.log(b === arguments[1]);
}
// true
// true
// false
// false

function fn(a, b = 3){
  console.log(arguments.length);
  console.log(a === arguments[0]);
  console.log(b === arguments[1]);
  a = 1; 
  b = 2;
  console.log(a === arguments[0]);
  console.log(b === arguments[1]);
}
// 1
// true
// false
// false
// false

                      • 默认参数值可以为函数。

                      • 默认参数值可以为其他命名参数(但是只能引用前面的参数)，这与临时死区有关。

                        • 一旦设置了参数的默认值，函数进行声明初始化时，参数会形成一个单独的作用域。如果声明在作用域的变量没有声明，那么它也会沿着作用域链寻找。
                        • 定义参数时，参数进入临时死区。只有当函数调用时，参数才进行初始化，从临时死区中移出。而错误的定义会使参数在被初始化之前被访问。

                      • 参数不得省略

                      function throwIfMissing() {
  throw new Error('Missing parameter');
}

function foo(mustBeProvided = throwIfMissing()) {
  return mustBeProvided;
}

foo()
// Error: Missing parameter

                      # rest 参数(不定参数)

                      • 引入不定参数，只能放在所有参数之后，不能放在setter中， 不影响arguments
                      • Function构造函数支持默认参数和不定参数。
                      • 是真正的数组

                      # 块级函数

                      定义在代码块内的函数

                      "use strict"
if(true){   
　　// 在ES5中抛出语法错误，ES6 中不报错；
　　function doSomething(){
　　　　//空函数
　　}
}
                      • 严格模式下，ES6可以在块级作用域中进行函数声明，并且存在"函数提升"。但是使用let 不会（函数表达式）。
                      • 非严格模式下，块级函数会提升至外围函数或全局作用域的顶部。（在代码块外部可访问）

                      # 箭头函数

                      • 特点：

                        • 没有this ，super，arguments，new.target 绑定
                        • 函数体内的this对象，就是定义时所在的对象，而不是使用时所在的对象。 this不可改变。且不能通过 call apply bind 改变this， 但仍可以借用其方法。
                        • 不可以当作构造函数，也就是说，不可以使用new命令，否则会抛出一个错误。
                        • 不可以使用arguments对象，该对象在函数体内不存在。如果要用，可以用 rest 参数代替。但可以访问外层arguments
                        • 不可以使用yield命令，因此箭头函数不能用作 Generator 函数。
                        • 箭头函数没有原型
                        • 返回对象时要加 （）

                      • 不适用场合：

                        • 定义对象方法，且方法内部有this
                        • 需要动态 this 的时候

                      # 尾调用

                      **尾调用：**在函数内部的最后一步（不一定是最后一句）是调用另一个函数

                      // 不是
function f(x){
  return g(x) + 1;
}

// 不是
function f(x){
  g(x);
}
// 等同于 
function f(x){
  g(x);
  return undefined;
}

// 是
function f(x){
  return g(x);
}

                      **尾调用优化：**只保留内层函数的调用帧

                      • 不存在闭包
                      • 尾调用是最后一步
                      • 尾调用结果作为函数值返回
                      • 只在严格模式下开启

                      可以用尾调用来优化递归
