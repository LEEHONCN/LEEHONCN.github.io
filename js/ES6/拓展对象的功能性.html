                      # 拓展对象的功能性

                      # 对象字面量的改动

                      • 对象的属性和本地变量重名时，可以简写。
                      module.exports = { getItem, setItem, clear };
// 等同于
module.exports = {
  getItem: getItem,
  setItem: setItem,
  clear: clear
};
                      • 对象方法可以简写，只有简写的对象方法才能使用super (也具有name属性)。

                      super.foo 等同于 Object.getPrototypeOf(this).foo（属性）或Object.getPrototypeOf(this).foo.call(this)（方法）

                      const obj = {
  foo: 'world',
  find() {
    return super.foo;
  }
};

// 报错
const obj = {
  foo: super.foo
}

// 报错
const obj = {
  foo: () => super.foo
}

// 报错
const obj = {
  foo: function () {
    return super.foo
  }
}
                      • 对象字面量中可以使用可计算属性名 例如 [name]。但是两种方法不能同时使用。
                      // 方法一
obj.foo = true;

// 方法二
obj['a' + 'bc'] = 123;
                      • 对象字面量中属性可以重复，以后面的为准。（会覆盖呗）
                      • 枚举顺序被定义
                        • 仅仅对 Object.getOwnPropertyNames() 和 Reflect.ownKeys 有效
                        • for .. in 和 Object.keys() 和 JSON.stringfy() 不明确
                        • 规则
                          • 所有数字键按升序排序
                          • 所有字符串和Symbol按照它们被加入对象的顺序排序

                      # 解构与拓展运算符

                      拓展运算符必须是最后一个参数

                      let { ...x, y, z } = someObject; // 句法错误
let { x, ...y, ...z } = someObject; // 句法错误

                      解构赋值是浅拷贝。

                      拓展运算符的解构赋值不能复制原型对象上的属性。

                      拓展运算符用在 整数，布尔型，undefined，null 会转为对象。由于这些对象没有自身属性所以返回的是空对象。

                      拓展运算符等效于 Object.assign({}, a)

                      # 可枚举性

                      enumerable：false 的属性不可枚举

                      • for...in循环：只遍历对象自身的和继承的可枚举的属性。
                      • Object.keys()：返回对象自身的所有可枚举的属性的键名。
                      • JSON.stringify()：只串行化对象自身的可枚举的属性。
                      • Object.assign()： 忽略enumerable为false的属性，只拷贝对象自身的可枚举的属性。

                      # 遍历

                      • for...in for...in循环遍历对象自身的和继承的可枚举属性（不含 Symbol 属性）。

                      • Object.keys(obj) Object.keys返回一个数组，包括对象自身的（不含继承的）所有可枚举属性（不含 Symbol 属性）的键名。

                      • Object.getOwnPropertyNames(obj)Object.getOwnPropertyNames返回一个数组，包含对象自身的所有属性（不含 Symbol 属性，但是包括不可枚举属性）的键名。

                      • Object.getOwnPropertySymbols(obj)Object.getOwnPropertySymbols返回一个数组，包含对象自身的所有 Symbol 属性的键名。

                      • Reflect.ownKeys(obj) Reflect.ownKeys返回一个数组，包含对象自身的所有键名，不管键名是 Symbol 或字符串，也不管是否可枚举。

                      # 新增方法

                      • Object.is() 接受两个参数，如果两个参数类型相同且具有相同的值，则返回true
                      • Object.assign() 可以接受任意数量的源对象，并按指定顺序将属性复制到接受对象中，如果多个源对象具有同名属性，会覆盖。

                      # 增强对象原型

                      • 改变对象原型 Object.setPrototypeOf() 修改其 [[prototype]] 指针 。
                      let man = Object.create(person);
console.log(Object.getPrototypeOf(man) === person);  // dog
Object.setPrototypeOf(man, dog);
console.log(Object.getPrototypeOf(man) === dog);     // dog

                      • 正式的方法定义

                        在ES6之前，对象的方法没有明确的定义。它仅仅是一个具有功能且没有数据的一个对象属性。

                        一个函数，它内部有一个[[HomeObject]] 指针指向这个方法从属的对象。

                      • Super 指向对象原型的指针。 只能在简写对象的方法里使用。Super 引用的运行过程。

                        • 在[[HomeObject]]属性上调用Object.getPrototypeOf() 来检索原型的引用。// 即获取该方法对象的原型对象
                        • 在原型对象中，找到同名方法。 // 很好理解
                        • 设置this绑定并且调用相应的方法。 // 用指向该对象的this替换原型对象方法的this。 类似于object.methodName.bind(this)
