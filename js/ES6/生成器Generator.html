                      # 生成器Generator

                      生成器：

                      是一种返回迭代器的函数。生成器即使迭代器，又是可迭代对象。

                      调用 Generator 函数后，该函数并不执行，返回的也不是函数运行结果，而是一个指向内部状态的指针对象，也就是迭代器对象（Iterator Object）。以后，每次调用遍历器对象的next方法，就会返回一个有着value和done两个属性的对象。value属性表示当前的内部状态的值，是yield表达式后面那个表达式的值；done属性是一个布尔值，表示是否遍历结束。认为迭代器对象是生成器的实例，继承了 Generator 函数的prototype对象上的方法。

                      • 函数中使用yield，但不能在Generator函数中的函数使用
                      • 通过在function关键字和小括号中添加 (*)号，不能用箭头函数创建
                      • 可以成为对象的方法

                      可以使用Generator并不使用yield 来单纯用作惰性执行。

                      **yield *：**其实就是解决 生成器嵌套问题

                      function* bar() {
  yield 'x';
  yield* foo();
  yield 'y';
}

// 等同于
function* bar() {
  yield 'x';
  yield 'a';
  yield 'b';
  yield 'y';
}

// 等同于
function* bar() {
  yield 'x';
  for (let v of foo()) {
    yield v;
  }
  yield 'y';
}

                      同样的，只要 Iterator 接口，都能被遍历。

                      let read = (function* () {
  yield 'hello';
  yield* 'hello';
})();

read.next().value // "hello"
read.next().value // "h"

                      如果内部生成器有返回值，那么外部生成器可以接收返回值。

                      利用 **yield *** 可以很好地进行二叉树的遍历。

                      next：

                      • 第一个next用来开始执行

                      • next方法的参数作为 上一个 yield 的返回值。

                      • 遍历器对象的next方法的运行逻辑如下：

                        • 遇到yield表达式，就暂停执行后面的操作，并将紧跟在yield后面的那个表达式的值，作为返回的对象的value属性值。
                        • 下一次调用next方法时，再继续往下执行，直到遇到下一个yield表达式。
                        • 如果没有再遇到新的yield表达式，就一直运行到函数结束，直到return语句为止，并将return语句后面的表达式的值，作为返回的对象的value属性值。
                        • 如果该函数没有return语句，则返回的对象的value属性值为undefined。

                      Generator.prototype.throw()：

                      • 可在生成器内部捕获，也可以在生成器外部捕获。

                      • 可以向 throw 传入参数，该参数会被catch 接收。

                      • 如果next一次都没有执行过，那么会被 外部 catch 捕获。

                      • 调用 throw 后会执行一次 next。

                      • 如果没有被捕获，那么就终止程序。

                      Generator.prototype.return()：

                      • 返回参数的值，并终止遍历。（之后next的输出都是 {value:undefined, done: true}）
                      • 如果生成器中有finally 代码块。那么会执行完 finally 再执行return 方法。

                      next throw return 共同点：

                      • next()是将yield表达式替换成一个值。

                      • throw()是将yield表达式替换成一个throw语句。

                      • return()是将yield表达式替换成一个return语句。

                      生成器中的this：

                      • Generator 函数总是返回迭代器，并且继承目标生成器的原型对象的方法。

                      • 不能作为构造函数

                      • 将 this 与 返回的对象统一

                        function* gen() {
  this.a = 1;
  yield this.b = 2;
  yield this.c = 3;
}

function F() {
  // 执行gen()  并将 this 绑定 gen.prototype
  return gen.call(gen.prototype);
}
// 获取迭代器对象
var f = new F();
// 执行 生成器原型方法
f.next();  // Object {value: 2, done: false}
f.next();  // Object {value: 3, done: false}
f.next();  // Object {value: undefined, done: true}
// 获取 生成器属性
f.a // 1
f.b // 2
f.c // 3

                      应用场景：

                      • 作为状态机结构
                      var clock = function* () {
  while (true) {
    console.log('Tick!');
    yield;
    console.log('Tock!');
    yield;
  }
};

                      • 异步函数同步化表达

                      • 控制流管理

                      • 部署 Iterator 接口

                      异步应用：

                      var fs = require('fs');

var readFile = function (fileName){
  return new Promise(function (resolve, reject){
    fs.readFile(fileName, function(error, data){
      if (error) return reject(error);
      resolve(data);
    });
  });
};

var gen = function* (){
  var f1 = yield readFile('/etc/fstab');
  var f2 = yield readFile('/etc/shells');
  console.log(f1.toString());
  console.log(f2.toString());
};

var g = gen();
g.next().value.then(function(data){
  // 每一层一个任务
  g.next(data).value.then(function(data){
    g.next(data);
  });
});

                      在异步任务中我们可以看出生成器的操作较为不便，不但每次都需要手动调用 next()，而且还需要知道什么时候调用，管理起来非常复杂。我们可以通过 co 模块来实现自动执行。async 和 await 就是 生成器和自动执行的语法糖。

                      co 模块：

                      function co(gen) {
  // 保存当前的执行环境
  var ctx = this;
  // 切割出函数调用时传递的参数
  var args = slice.call(arguments, 1)
  return new Promise(function(resolve, reject) {
    if (typeof gen === 'function') gen = gen.apply(ctx, args);
    if (!gen || typeof gen.next !== 'function') return resolve(gen);

    // 首次调用gen.next()函数，假如存在的话
    onFulfilled();

    function onFulfilled(res) {
      var ret;
      try {
        // 尝试着获取下一个yield后面代码执行后返回的值
        ret = gen.next(res);
      } catch (e) {
        return reject(e);
      }
      // 处理结果
      next(ret);
    }

    /**
     * @param {Error} err
     * @return {Promise}
     * @api private
     */

    function onRejected(err) {
      var ret;
      try {
        // 尝试抛出错误
        ret = gen.throw(err);
      } catch (e) {
        return reject(e);
      }
      // 处理结果
      next(ret);
    }

    /**
     * Get the next value in the generator,
     * return a promise.
     *
     * @param {Object} ret
     * @return {Promise}
     * @api private
     */

    // 这个next()函数是最为关键的一部分，
    // 里面几乎包含了generator自动调用实现的核心
    function next(ret) {
      // 如果ret.done === true, 
      // 证明generator函数已经执行完毕
      // 即已经返回了值
      if (ret.done) return resolve(ret.value);
      
      // 把ret.value转换成Promise对象继续调用
      var value = toPromise.call(ctx, ret.value);
      
      // 如果存在，则把控制权交给onFulfilled和onRejected，
      // 实现递归调用
      if (value && isPromise(value)) return value.then(onFulfilled, onRejected);
      
      // 否则最后直接抛出错误
      return onRejected(new TypeError('You may only yield a function, promise, generator, array, or object, '
        + 'but the following object was passed: "' + String(ret.value) + '"'));
    }
  });
}
