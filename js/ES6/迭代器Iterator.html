                      # 迭代器Iterator

                      迭代器：

                      一种特殊对象，它具有一些专门为迭代过程设计的专有接口。next()方法返回一个结果对象({value, done})。保存内部指针，用来指向当前集合中值的位置。

                      总的来说，就是有next约定接口的对象。

                      可迭代对象和for-of循环：

                      • 生成器默认会为Symbol.iterator属性赋值，通过生成器创建的迭代器都是可迭代对象。该对象的[Symbol.iterator]指针指向自身。
                      • for-of循环每执行一次都会调用可迭代对象中的next()方法。将返回对象的value储存在变量中，直到返回对象的done属性的值为true。
                      • [Symbol.iterator]
                        • 可以通过[Symbol.iterator]来访问对象默认的迭代器。
                        • 也可以通过[Symbol.iterator]来检测是否为可迭代对象。
                        • 也可以通过[Symbol.iterator] 添加一个生成器，使其变为可迭代对象。
                        • 自然也可以修改咯。

                      内建迭代器：

                      数组，Map集合，Set集合三种对象都内建了以下三种迭代器

                      • entires() Map集合默认的迭代器。
                      • values() 数组和Set集合默认的迭代器。
                      • keys()
                      • 其实默认迭代器就是 [Symbol.iterator]原本指向的生成器。

                      原生具备 Iterator 接口的数据结构如下：

                      Array Map Set String TypedArray 函数的 arguments 对象 NodeList 对象

                      展开运算符的运算过程：

                      • 根据默认迭代器（[Symbol.iterator]指向的生成器）来选取要引用的值。
                      • 从迭代器读取所有值。
                      • 然后按照返回顺序将它们依次插入到数组中。

                      高级迭代器功能：

                      • 可以给迭代器的next()方法传递参数，则这个参数的值就会替代生成器内部的上一条yield语句的返回值。
                        • 通过该功能可以使对象在处于不同状态。
                      • 在迭代器中抛出错误 Generator.prototype.throw()
                      • 生成器返回语句 Generator.prototype.return()
                      • 生成器委托 yield *
