          # 事件

          # 事件流

          # 事件冒泡

          事件开始由最具体的元素接受，然后逐级向上传播到较为不具体的节点。

          # 事件捕获

          不太具体的节点应该更早接受到事件，具体的节点应该最后接受到事件。

          # Dom事件流

          事件捕获 处于目标 事件冒泡

          # 事件处理程序

          事件委托：利用事件冒泡机制，可以只用一个事件处理程序完成一类事件的处理。

          # HTML事件处理程序

          # Dom0级事件处理程序

          使用 DOM0 级方法指定的事件处理程序被认为是元素的方法。因此，这时候的事件处理程序是在 元素的作用域中运行，因此 函数中的this为element对象。

          在事件冒泡阶段处理

          let btn = document.getElementById("myBtn");
btn.onclick = function(){
  alert(this.id);
}

          # Dom2级事件处理程序

          true 事件捕获阶段 false 事件冒泡阶段。

          一般来说在事件冒泡阶段进行捕获，除非有特定需求。

          let btn = document.getElementById("myBtn");
btn.addEventListenr("click", function(){
  alert("Hello world!");
}, false);

btn.removeListener("click", function(){
  alert("Hello world!");
}, false);

          # 事件对象

          浏览器会将一个event对象传入事件处理程序中，无论是 Dom0 还是 Dom2

          # 属性和方法

          属性/方法 类型 说明
          bubbles Boolean 事件是否冒泡
          cancelable Boolean 表明是否可以取消事件的默认行为
          defaultPrevented Boolean true 说明已经调用了 preventDefault()
          trusted Boolean 是浏览器事件还是自定义事件
          detail Integer
          eventPhase Integer 调用事件处理程序的阶段
          type String 被触发事件的类型
          preventDefault() Function 取消事件的默认行为，cancelable为true可以调用
          stopImmediatePropagation() Function 取消事件进一步捕获或冒泡，同时阻止事件处理程序被调用
          stopPropagation() Function 取消事件进一步捕获或冒泡
          currentTarget Element 事件正在处理事件的那个元素
          target Element 事件的目标
          view AbstractView

          如果click事件的目标是按钮，则三值相等（currentTarget总是等于this）。如果click事件的处理程序在按钮父节点，则currentTarget与target不同。

          var btn = document.getElementById("myBtn");
btn.onclick = function(event){
    alert(event.currentTarget === this);  // true
    alert(event.target === this);         // true
};


document.body.onclick = function(event){ 
  alert(event.currentTarget === document.body); //true 
  alert(this === document.body);                //true 
  alert(event.target === document.getElementById("myBtn"));    // true
};

          # 实践

          • Type：利用type在一个函数中处理多个事件

          • preventDefault()：取消默认行为

          • stopPropagation()：取消事件传播

          # 事件类型

          # UI事件

          事件名 描述
          load 当页面完全加载后在 window 上面触发，当所有框架都加载完毕时在框架集上面触发， 当图像加载完毕时在元素上面触发
          unload
          abort
          error
          select
          resize 当窗口或框架的大小变化时在 window 或框架上面触发。
          scroll 当用户滚动带滚动条的元素中的内容时，在该元素上面触发。元素中包含所加载页面的滚动条。

          # 焦点事件

          事件名 描述
          blur 在元素失去焦点时触发，不会冒泡
          focus 在元素获取焦点时触发，不会冒泡
          focusin 在元素获取焦点时触发，冒泡
          focusout 通用版blur

          # 鼠标与滚轮事件

          事件名 描述
          click 在用户单击主鼠标按钮(一般是左边的按钮)或者按下回车键时触发
          dbclick 在用户双击主鼠标按钮(一般是左边的按钮)时触发
          mousedown 在用户按下了任意鼠标按钮时触发。不能通过键盘触发这个事件
          mouseup 在用户释放鼠标按钮时触发
          mouseenter 在鼠标光标从元素外部首次移动到元素范围之内时触发。这个事件不冒泡
          mouseleave 在位于元素上方的鼠标光标移动到元素范围之外时触发。这个事件不冒泡
          mousemove 当鼠标指针在元素内部移动时重复地触发
          mouseout 在鼠标指针位于一个元素上方，然后用户将其移入另一个元素时触发
          mouseover 在鼠标指针位于一个元素外部，然后用户将其首次移入另一个元素边界之内时触
          发。
          mousewheel
