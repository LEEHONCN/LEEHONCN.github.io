          # 原型链，变量，作用域，内存问题

          原型链图示

          由图可总结

          • 实例的 _proto_ 指向 构造函数的原型。 构造函数的 prototype 指向其原型。

          • 所有函数的 _proto_=== Function.prototype 包括 Object() , Number()。

          • Function.prototype._proto_ === Object.prototype

          • 无继承的构造函数的 prototype._proto_ === Object.prototype

          • **Object.protoype._proto_ === null **

          其中 prototype 也可以是实例对象，由此构成的链条为原型链。注意 2，3，4有传递性。

          new 操作符创建对象的过程

          • 创建一个新对象
          • 将对象的 _proto_ 链接到构造函数的 prototype 属性上。
          • 绑定 this ，执行构造函数。
          • 返回新对象

          instanceof ：根据原型链进行类型检查，因此只能检查引用类型。

          全局环境：所有变量和函数都是作为全局执行环境(window对象)的属性和方法创建的。

          执行上下文生命周期(步骤)：

          • 创建阶段
            • 创建变量对象
            • 创建作用域链
            • 确定this的指向
          • 执行阶段
            • 执行环境入栈
            • 变量对象 -> 活动对象
            • 执行代码
          • 执行完毕
            • 执行环境出栈

          变量对象(VO) ： 每一个执行环境都有一个与之关联的变量对象，环境中定义的所有变量和函数都保存在这个对象中。其包含：参数，变量，函数。

          变量对象的创建过程：

          1. 建立arguments对象，作为变量对象的属性。若未传入，arguments为undefined。
          2. 检查函数声明，并在变量对象中建立属性，属性值指向函数的内存地址（若命名冲突则覆盖）。 ---------- 函数提升，但此函数已被初始化
          3. 检查变量声明，如果发送命名冲突则忽略，否则为变量对象创建一个属性，其值为undefined。 —-------- 变量提升，只是声明，没有被初始化。

          活动对象(AO) ：变量对象不能被访问，只有转为活动对象后才能被访问。

          执行流的机制：每个函数都有自己的执行环境，当执行流进入一个函数时，函数的环境就会被推入一个环境栈，执行结束后，栈将该执行环境弹出。

          作用域链：当代码在一个环境中执行时，会创建变量对象的一个作用域链，作用域链的用途，是保证对执行环境有权访问的所有变量和函数的有序访问。通常的arguments对象作为作用域链的第一个对象，全局执行环境的变量对象作为作用域链中的最后一个对象。

          标识符解析：标识符解析(变量名解析)是沿着作用域链进行的，从前往后，找到即停止。

          延长作用域 : with 将指定对象添加到作用域链中， catch 会创建新的变量对象

          没有块级作用域 ：ES6中加入，也可以用立即执行来模仿块级作用域。

          this：它代表函数运行时，自动生成的一个内部对象。执行时才能确认值，定义时无法确认

          • 构造函数：指向新对象
          • 对象成员方法：指向所属对象
          • 普通方法：指向全局对象
          • call apply bind：指向绑定对象

          闭包： 有权访问另一个函数作用域中的变量的函数。

          内部函数会将外部函数中的活动对象添加到它的作用域链中。

          闭包因携带包含其的函数的作用域，它占用的内存比其他函数更多。

          作用：封装变量外部无法访问，变量保存在内存中

          垃圾收集： 标记清除，引用计数。 最佳 通过将变量赋值为null解除引用。

          总结：

          Js作用域 的主要概念是，执行上下文 变量对象 作用域链 。执行上下文描述了Js引擎在创建到执行完毕的过程。其中创建阶段完成变量对象的创建和作业域链的，这也是变量提升和函数提升的原因(let和const不能提升是因为暂时锁区)。执行阶段，将执行环境入栈，变量对象转为活动对象因此可以执行，通过作用域链访问外层变量和函数。
