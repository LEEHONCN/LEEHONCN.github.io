          # 基于原型的面向对象系统

          # 创建对象

          • 工厂模式

            • 存在的问题:虽然创建了多个对象。但是无法识别对象的类型。

          • 构造函数模式 所有放在构造函数内部

            • new 操作符的执行步骤
              • 创建一个对象
              • 将构造函数的作用域赋给新变量
              • 执行构造函数中的代码
              • 返回新对象
            • 存在的问题:可以识别对象的类型，但是无法复用成员变量和方法。

          • 原型模式 所有放在原型对象上。

            //  创建这个函数时，该函数会创建一个prototype属性，指向其原型对象。
//  其原型对象会创建constructor属性指向构造函数
function Person(){
}
// 为原型对象添加属性和方法。
Person.prototype.name = "Nicholas";
Person.prototype.age = 29;
Person.prototype.job = "Software Engineer";
Person.prototype.friends = ["aa", "bb"];
Person.prototype.sayName = function(){
  alert(this.name);
};

var person1 = new Person();
person1.sayName();   //"Nicholas"
person1.friends.push("cc");
var person2 = new Person();
person2.sayName();   //"Nicholas"

console.log(person1.sayName == person2.sayName);  //true
console.log(person1.friends == person2.friends);  //true
            • 存在的问题:共享引用类型的变量会导致，一个实例的修改影响另一个实例。

          • 组合使用构造函数模式和原型模式(推荐)

            使用构造函数定义实例属性，而原型模型用于定义方法和共享属性。

          function Person(name, age, job){
    this.name = name;
    this.age = age;
    this.job = job;
    this.friends = ["Shelby", "Court"];
}

Person.prototype = {
    constructor: Person,
    sayName : function () {
        alert(this.name);
    }
};

let person1 = new Person("Nicholas", 29, "Software Engineer");
let person2 = new Person("Greg", 27, "Doctor");

person1.friends.push("Van");

console.log(person1.friends);    //"Shelby,Court,Van"
console.log(person2.friends);    //"Shelby,Court"
console.log(person1.friends === person2.friends);  //false
console.log(person1.sayName === person2.sayName);  //true
          • 动态原型模式(推荐)
          function Person(name, age, job){
    //properties
    this.name = name;
    this.age = age;
    this.job = job;
		//methods
  	// 只有在初次调用构造函数时才执行。对于if，只写一个就行。
		if (typeof this.sayName != "function"){
        Person.prototype.sayName = function(){
          alert(this.name);
        };
		}
}

var friend = new Person("Nicholas", 29, "Software Engineer");
friend.sayName();

          总结：

          • 构造函数模式，不能复用“父类”的成员变量和方法

          • 原型模式，可以复用但是变量共享同一块内存（仅仅是从原型链上查找）

          • 组合模式，完美解决问题，共享的变量或方法放在构造函数里，不共享的放在原型上。

          • 动态原型模式，同理只不过将原型方法的赋值写在构造函数里。

          # 继承

          ​ 当一个原型对象是另一个原型对象的实例，该原型对象就拥有了一个指向另一个原型对象的指针[[prototype]]。这样就构成了基于原型的继承 — 原型链。

          • 原型链继承的问题:由于一个原型对象的实例也是原型对象，所以该原型对象的实例属性被"继承”为原型属性。这样又引起了共用引用类型导致的实例之间的相互影响。
          function SuperType(){
  this.colors = ["red", "blue", "green"];
}

function SubType(){            
}

//inherit from SuperType
SubType.prototype = new SuperType();

var instance1 = new SubType();
instance1.colors.push("black");
alert(instance1.colors);    //"red,blue,green,black"

var instance2 = new SubType();
alert(instance2.colors);    //"red,blue,green,black"
          • 借用构造函数方法，导致方法无法复用。
          • 组合继承(常用）与创建对象中相似，把方法定义在原型上而不是构造函数上。但是无论什么情况下， 都会调用两次超类型构造函数。
          function SuperType(name){
    this.name = name;
    this.colors = ["red", "blue", "green"];
}

SuperType.prototype.sayName = function(){
    console.log(this.name);
};

function SubType(name, age){
    SuperType.call(this, name);   // 1    将原型链上父类的属性转移到子类上来。
    this.age = age;
}

SubType.prototype = new SuperType();   // 2    
SubType.prototype.sayAge = function(){
    console.log(this.age);
};

instance1.colors.push("black");
console.log(instance1.colors);  //"red,blue,green,black"
instance1.sayName();      //"Nicholas";
instance1.sayAge();       //29

          • 原型式继承

          使用对象而不是构造函数创建对象，同时也继承了该对象的所有属性和方法。 Object.create() 或 object() 可以实现。

          Object.create = function (obj){
		let F = function(){}
    F.prototype = obj
  	return new F();
}

          也就是说，它只能用父类的属性和方法。

          • 寄生式继承

          并不是真正的继承，通过将传入的对象原型式继承来创建对象，再封装了增强(为对象添加方法)，最后在返回。这样它本身也算有一些自己的方法了。

          • 寄生组合式继承(最佳) 超类型的构造函数只执行一次。
          function inheritPrototype(subType, superType){
  	// 创建超类型的实例
    let prototype = Object.create(superType.prototype);   //create object
  	// 将超类型的实例的构造方法设置为子类型的构造方法
    prototype.constructor = subType;                      //augment object
  	// 将子类型的原型对象设置为超类型的实例
    subType.prototype = prototype;                        //assign object
}
// 超类型的构造方法
function SuperType(name){
    this.name = name;
    this.colors = ["red", "blue", "green"];
}

// 子类型的构造方法
function SubType(name, age){
    SuperType.call(this, name);
    this.age = age;
}

// 为超类型的原型对象添加方法
SuperType.prototype.sayName = function(){
    console.log(this.name);
};

// 为子类型的原型对象添加方法
SubType.prototype.sayAge = function(){
    console.log(this.age);
};

// 将子类型"继承"于超类型 
inheritPrototype(SubType, SuperType);

let instance1 = new SubType("Nicholas", 29);
instance1.colors.push("black");
console.log(instance1.colors);  //"red,blue,green,black"
instance1.sayName();      //"Nicholas";
instance1.sayAge();       //29

let instance2 = new SubType("Greg", 27);
console.log(instance2.colors);  //"red,blue,green"
instance2.sayName();      //"Greg";
instance2.sayAge();       //27

          总结：

          • 原型链继承、构造函数继承、和组合继承：与前边创建对象的问题相似。但是组合继承不是最佳实践，因为它要调用两次超类的构造方法。
          • 原型式继承就是用 Object.create()来创建子类。用的全是父类的方法和属性。
          • 寄生式继承，就是 创建父类对象 + 封装增强对象并作为子类对象 。有一些自己的方法和属性。
          • 寄生式组合继承就是，将父类的子类的构造函数的原型连接起来，并用call方法将父类的属性复制到子类上，并可以在父类和子类的原型上添加方法。 这样子类有全部的属性，并可以在父类原型上添加一些公有方法。并且只需要调用一次父类的构造函数。
