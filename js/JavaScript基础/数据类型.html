          # 数据类型

          # 数据类型以及区分

          JavaScript有7种数据类型

          • 数值 number
          • 字符串 string
          • 布尔值boolean
          • 未定义 undefined
          • 空值 null
          • 对象 object
            • 狭义的对象 object
            • 数组 array
            • 函数 Function
            • 其他对象
          • 符号 Symbol

          区分类型

          • typeof 不能区分 null，object，array，所有种类的Object（全都返回 object）
          • instanceof 可以区分 array 和 object 。也就是可以区分所有种类的Object

          null和undefined

          • null转为数值时为 0，而undefined转为数值时为 NaN
          • undefined出现的时刻：
            • 未赋值
            • 未传递参数
            • 没有返回值

          # 类型转换的情况

          类型转为布尔值

          条件判断时，类型转换为false 的情况： undefined null false 0 NaN ""或''（空字符串）

          特例: [] {}

          简单记，除了Symbol和Object所有基本类型来一遍。数字类型有两个 0 和 NaN

          对象转基本类型

          调用优先级 Symbol.toPrimitive valueOf toString

          四则运算符

          + 有一方是字符串类型，那么另一个要转为字符串类型。并且会触发三种类型转换：转为原始值，数字，字符串。比如

          console.log(typeof (1 + new Set()));   // string

          其他运算符 都是有一方为数字就转为数字，那么经常是 NaN。

          console.log((1 / new Set()));   // NaN

          == 操作符

          • 若两个类型相同
            • undefined null 返回 true
            • 对象引用相同 返回 true
            • 比较字符串，完全相同的字符串序列 返回 true
            • 布尔类型就好说了
            • 数字类型：若 相等数值 或 +0和 -0 返回 true
            • Symbol 使用 Symbol.for 注册过，并且Key 值相同的两个Symbol对象 返回 true
          • undefined 和 null undefined和undefined null 和 null 返回 true 其他任何与这两种比较都为 false
          • String 和 Number 会将字符串转为数字再比较
          • Boolean 和 Number 会将布尔值转为数字再比较
          • 若 对象 和 字符串或数字比较 会先调用 toPrimitive
          • Symbol 和 其他所有类型比较都为 false

          比较运算符

          如果是对象，就通过 toPrimitive 转换对象

          如果是字符串，就通过 unicode 字符索引来比较

          # 一些类型的特性

          字符串

          字符串的取值也可以像数组那样，它也有length属性，但是使用方括号运算符赋值以及修改length的值都无效。

          多行字符串： `` ， 字符串相加， 函数注释转字符串切分， \

          所有字符都用Unicode形式表示，但是存储使用 UTF-16。因此，当出现Unicode的编码大于16bit即OxFFFF时，一个字符的长度为2。

          Base64编码：为了避免特殊字符，简化程序处理。

          • btoa()：任意值转为 Base64 编码
          • atob()：Base64 编码转为原来的值

          对象

          in：检查属性是否存在，包括继承得来的。可通过obj.hasOwnProperty()排除继承的属性。

          for in：遍历所有可遍历的属性，包括继承的。

          with：将对象作为作用域

          函数

          eval命令，接受字符串，并将字符串当做语句运行。

          严格模式下，eval声明的变量不会影响到外部，但是可以修改外部变量。

          别名调用(非直接调用)，字符串的执行环境都是全局作用域。

          数组

          数组的空位可以读取，也计入length的值。会被 forEach，for in ， Object.keys 跳过。
