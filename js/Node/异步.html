# 异步

# Js的单线程模型与任务队列和事件循环

主线程负责执行串行脚本，其他线程负责异步任务（具有回调函数）。

Js提供任务队列，里面存放着需要处理的异步任务。

当主线程的同步任务处理完毕后，通过事件循环机制（循环检查）去任务队列中寻找满足条件的异步任务，并加入主线程执行其回调函数。

微任务直接加入本次事件循环，宏任务加入下次事件循环

事件循环机制

  • 事件循环机制有三个主要部分：主线程执行堆栈(stack)，定时器处理器(handler)，任务队列(queue)。

  • 首先将代码依次进入stack并执行，如果遇到setTimeout这样的定时器，则将回调任务加入到handler。

  • 当任务进入handler时开始计时，一旦到达设定时间，handler负责任务加入queue。

  • 当stack空闲时，会执行queue中的任务。（其中微任务早于宏任务被清空，或者说先清空微任务队列，再清空宏任务队列）

  • queue为空时，queue等待主线程，进入下一个事件循环。

宏任务包括： setTimeout, setInterval, setImmediate, I/O, UI rendering。

微任务包括：process.nextTick, Promises, Object.observe, MutationObserver。

如果一个微任务不断地产生这个微任务，那么会发生阻塞。例：

function foo() {
  return Promise.resolve().then(foo);
};

# 定时器API

var timerId = setTimeout(func|code, delay(ms),...argus); //回调函数或语句，延时毫秒数，传入函数的参数，返回定时器的计数可以用来清除该定时器 
var timerId2 = setInterval();   // 参数,返回值同setTimeout。但是每隔一段时间执行一次。 
clearTimeout(timerId);      
clearInterval(timerId2);
# 注意

setInterval 指的是开始执行之间的时间。如果执行时间大于延时时间，则立即执行。配合 setTimeout可以做到 确定本次结束执行时间与下次执行开始的时间间隔。

setTimeout和setInterval的运行机制，是将指定的代码移出本轮事件循环，等到下一轮事件循环，再检查是否到了指定时间。如果到了，就执行对应的代码；如果不到，就继续等待。它们是宏任务。

setTimeout(func,0);    // 并不是立即执行，而是把该函数放入下一个事件循环的首位。
setTimeout(func,100);  // 实际的延时时间 大于等于 100ms
# 应用场景
  • debounce函数 (防抖动函数)
  • setTimeout(func,o);
  • 可以调节事件发生顺序
  • 调整用户自定义的回调函数与浏览器默认动作之间的触发顺序(如果回调函数于浏览器的话)。
  • 将计算量大的任务分散，避免长时间占用主线程。
