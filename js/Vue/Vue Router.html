    # Vue Router

    将组件映射得到路由，然后告诉 Vue Router在哪里渲染它们。

    当 router-link 对应的路由匹配成功，将自动设置 class 属性值为 .router-link-active

    <router-link to="/foo">Go to Foo</router-link>
<router-link to="/bar">Go to Bar</router-link>
<router-view></router-view>

    设置路由映射的组件

    const routes = [
  { path: '/foo', component: Foo },
  { path: '/bar', component: Bar }
]

    通过 this.$router 访问路由器

    通过 this.$route 访问路由

    # 动态路由匹配

    const router = new VueRouter({
  routes: [
    // 动态路径参数 以冒号开头
    { path: '/user/:id', component: User
     	name: 'user'  // 命名路由，之后 router.push 就可以使用了。
    }
  ]
})

    可以通过 this.$route.params.id 来获取。

    当路由从 /user/foo 导航到 /user/bar 时组件会被复用。此时组件的生命周期钩子不会被调用。可以使用

    const User = {
  template: '...',
  watch: {
    '$route' (to, from) {
      // 对路由变化作出响应...
    }
  },
  beforeRouteUpdate (to, from, next) {  // 导航守卫
    // react to route changes...
    // don't forget to call next()
  }
}

    可以使用通配符

    {  path: '*' // 会匹配所有路径  
}
{  path: '/user-*'  // 会匹配以 `/user-` 开头的任意路径
}

    匹配优先级按照路由的定义顺序。

    # 嵌套路由

    需要配置 children

    const router = new VueRouter({
  routes: [
    { path: '/user/:id', component: User,
      children: [
        // 当 /user/:id 匹配成功，
        // UserHome 会被渲染在 User 的 <router-view> 中
        { path: '', component: UserHome },
        // ...其他子路由
      ]
    }
  ]
})

    # 编程式导航

    这些API的风格类似 window.history

    # router.push
    // 字符串
router.push('home')
// 对象
router.push({ path: 'home' })
// 命名的路由
router.push({ name: 'user', params: { userId: '123' }})
// 带查询参数，变成 /register?plan=private
router.push({ path: 'register', query: { plan: 'private' }})

const userId = '123'
router.push({ name: 'user', params: { userId }}) // -> /user/123
// 如果有 path 则不能有  params
router.push({ path: `/user/${userId}` }) // -> /user/123
    # router.replace

    与 push 相似，但是没有 history 记录。会替换掉当前的记录。

    # router.go

    在 history 记录中向前或者后退多少步

    # 命名视图

    想要同级展示多个视图，而不是嵌套显示。

    <router-view class="view one"></router-view>
<router-view class="view two" name="a"></router-view>
<router-view class="view three" name="b"></router-view>

    const router = new VueRouter({
  routes: [
    {
      path: '/',
      components: {
        default: Foo,
        a: Bar,
        b: Baz
      }
    }
  ]
})

    # 重定向与别名

    const router = new VueRouter({
  routes: [
    { path: '/a', redirect: '/b' },  // 重定向
    { path: '/a', component: A, alias: '/b' } // 别名
  ]
})

    总结：

    通过 VueRouter 对象配置 路由与组件的对应关系

    可以使用动态路由匹配来获取参数

    路由嵌套可以显示多层不同的内容。

    命名视图可以让同级展示多个视图。

    嵌套路由可以和命名视图结合使用。

    编程式导航可以通过代码改变URL。

    利用props 可以将路由和组件解耦。

    路由可以使用 H5的 HIstory模式。

    可以利用路由守卫来获取数据。

    可以使用Vue的过渡功能

    可以定义切换路由时，滚动到什么位置。

    可以将组件按组分块，实现组件懒加载。
