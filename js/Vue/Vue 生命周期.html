    # Vue 生命周期

    想要深入理解Vue官网生命周期图，我们可以通过阅读源代码进行理解。

    下面是Vue这个“类”的声明，可以看到当我们创建Vue对象时，这个**_init** 方法将被执行。最下面几个Mixin 为 Vue原型添加方法。 代码见 src/core/instance/index.js

    function Vue (options) {
  if (process.env.NODE_ENV !== 'production' &&
    !(this instanceof Vue)
  ) {
    warn('Vue is a constructor and should be called with the `new` keyword')
  }
  // options为我们传入的参数。即我们写Vue时，写的那个巨大的对象字面量。
  // 该方法在initMixin被装载
  this._init(options)
}

initMixin(Vue);
stateMixin(Vue);
eventsMixin(Vue);
lifecycleMixin(Vue);
renderMixin(Vue);
    # beforeCreate && created

    先看看这些Mixin

    # initMixin

    这个方法就是为 Vue 原型添加 _init 方法。这个方法的工作主要是两件事。

    1. 使用策略对象合并父子组件实例参数选项

    2. 初始化工作与Vue实例对象的设计，以及调用用户定义的生命周期钩子

      initLifecycle(vm)    // 初始化生命周期有关的属性
initEvents(vm)			// 初始化事件有关的属性
initRender(vm)			// 初始化渲染有关的属性
callHook(vm, 'beforeCreate')     // 调用用户定义的生命周期钩子
initInjections(vm) // resolve injections before data/props
initState(vm)			// 完成了  `data` ,`props` ,`methods`,`computed`,`watch`的初始化工作。数据绑定也是这个阶段完成的。
initProvide(vm) // resolve provide after data/props
callHook(vm, 'created')  // 调用用户定义的生命周期钩子

      以 initLifecycle 为例，初始化一些生命周期有关的属性。

    # beforeMount && mounted

    在 src/platforms/web/runtime/index.js 和 src/platforms/weex/runtime/index.js 下都挂载了 Vue.prototype.$mount 方法，该方法就是 core/instance/lifecycle 下的 mountComponent 方法。该方法完成组件的挂载工作并执行用户定义的生命周期函数。

    # beforeUpdate && updated

    前边已经说了 **beforeUpdate ** 在 _update 方法中，实际上它在 _update 方法中被放入一个 before 函数中并传给了 Watcher 也就是说每当 Watcher 调用了 before 方法 Vue实例的钩子就会被触发。

    那么 updated 呢。同样的，它被 import 进 Watcher 的 update 方法。并且通过 scheduler 保证 子组件的 updated钩子在父组件updated钩子之中。

    前边说了，在 Vue.prototype.$destroy 中。

    # activated & deactivated

    只有组件被 <keep-alive> 包裹时才能使用，用户定义的生命周期钩子分别在 src/core/instance/lifecycle.js 的 activateChildComponent 和 deactivateChildComponent 方法中被调用。分别在动态组件激活时，动态组件停用时被调用。

    # errorCaptured

    当捕获一个来自子孙组件的错误时被调用。此钩子会收到三个参数：错误对象、发生错误的组件实例以及一个包含错误来源信息的字符串。此钩子可以返回 false 以阻止该错误继续向上传播。

    # 总结：

    看看这些生命周期的特点：

    beforeCreate 仅仅初始化了一些生命周期，事件，渲染有关的属性。但是不能读取data，也没有进行数据绑定。

    Created 完成了 Injection，state，Provide的初始化，但还未挂载组件。可以在这个钩子修改state等，但是不能操作Dom元素，例如异步请求数据。

    beforeMount 在挂载开始之前被调用：相关的 render 函数首次被调用。服务器渲染期间不能使用。

    mounted 已经渲染，可以操作Dom元素。但是不能保证子组件也渲染完毕，可以使用 $nextTick 来保证这一问题。服务器渲染期间不能使用。

    beforeUpdate 数据更新，但是Dom没有渲染。适合操作数据更新前的Dom元素。服务器渲染期间不能使用。

    updated 数据更新 Dom已经渲染。同样地，不能保证子组件也重绘完毕。服务器渲染期间不能使用。

    **beforeDestroy ** 实例摧毁前，仍可用 服务器渲染期间不能使用。

    destroyed 实例摧毁后 服务器渲染期间不能使用。

    activated 动态组件激活时 服务器渲染期间不能使用。

    deactivated 动态组合停用时 服务器渲染期间不能使用。

    errorCaptured 子孙组件出现错误时，用于调试。

    # 相关API

    Vue.prototype.$mount 用于挂载为挂载的实例（未提供 el 属性）

    Vue.prototype.$forceUpdate 迫使实例进行重绘

    Vue.prototype.$destroy 摧毁一个实例

    Vue.prototype.$nextTick 接收一个回调函数，该函数会在下次Dom更新循环之后执行。
