    # Vuex

    vuex是一个专为vue.js应用程序开发的状态管理模式。

    为了解决组件之间同一状态共享的问题。

    只需要提取共享的状态，局部状态还是由组件保有。

    vuex有五种属性，分别是 State、 Getter、Mutation 、Action、 Module。

    State

    如何在组件中获取 Vuex状态： 在根组件注入 store 属性。该 store 就是 导出的 store。子组件中就可以通过 this.$store 访问到。mapState 可以用来简化混入到组件中的书写。

    Getter

    类似于计算属性。 同样 mapGetters可以用来简化混入组件的书写。

    Mutation

    使用 store.commit('Mutation名') 来调用 Mutation。

    // 提交载荷方式提交参数
store.commit('increment', {
  amount: 10
})
// 对象形式提交参数
store.commit({
  type: 'increment',
  amount: 10
})

mutations: {
  increment (state, payload) {
    state.count += payload.amount
  }
}

    要注意 Vue 响应式的规则，不能在 Mutation 中创建新的属性。

    Mutation 必须是同步函数。

    在组件中使用 this.$store.commit('xxx') 提交 mutation，或者使用 mapMutations 辅助函数将组件中的 methods 映射为 store.commit 调用（需要在根节点注入 store）。

    # Action

    使用 store.dispatch('Action名') 来调用 Action。 同样也可以通过 提交载荷和对象形式提交参数。

    在 Action 中可以进行一系列操作，并提交 Mutation 来记录 Action 产生的状态变更。并且 Action 可以进行异步操作。 Action 对应的是业务，而 Mutation 对应的是 一个属性。

    在组件中使用 this.$store.dispatch('xxx') 分发 action，或者使用 mapActions 辅助函数将组件的 methods 映射为 store.dispatch 调用（需要先在根节点注入 store）：

    # Module

    当项目结构复杂，尤其是可以提取出一部分有关系的状态（经常一起使用的）。这样的话项目结构就会很清晰。

    模块内部的 Mutation 和 getter 接收的第一个参数是模块的剧本状态对象。

    模块内部的 Action 局部状态通过 context.state 暴露出来，根节点状态则为 context.rootState

    模块内部的 getter 的第三个参数是根节点状态。

    默认情况下，模块内部的 action、mutation 和 getter 是注册在全局命名空间的——这样使得多个模块能够对同一 mutation 或 action 作出响应。可以通过添加命名空间，让模块有更好的封装性。

    触发时也需要添加命名空间前缀。

    getters['account/profile']
commit('account/login')
dispatch('account/login')

    也可以在命名空间中访问全局内容 如 Getter 和 Action 的第四参数。

    或调用 Action 或 Mutation 时加入 {root : true}

    dispatch('someOtherAction', null, { root: true }) 
commit('someMutation', null, { root: true })

    在模块中定义全局 action

    modules: {
    foo: {
      namespaced: true,
      actions: {
        someAction: {
          root: true,
          handler (namespacedContext, payload) { ... } // -> 'someAction'
        }
      }
    }
}

    命名空间与绑定函数

    ...mapActions([
    'some/nested/module/foo', // -> this['some/nested/module/foo']()
    'some/nested/module/bar' // -> this['some/nested/module/bar']()
  ])
// 等价于
 ...mapActions('some/nested/module', [
    'foo', // -> this.foo()
    'bar' // -> this.bar()
  ])
// 等价于 
const { mapState, mapActions } = createNamespacedHelpers('some/nested/module')
...mapActions([
      'foo',
      'bar'
    ])

    注册模块

    // 注册模块 `myModule`
store.registerModule('myModule', {
  // ...
})
// 注册嵌套模块 `nested/myModule`
store.registerModule(['nested', 'myModule'], {
  // ...
})
// 之后就可以访问
store.state.myModule
store.state.nested.myModule

    vuex的工作流程

    1. 在vue组件里面，通过dispatch来触发actions 业务流程操作。
    2. 然后再通过actions的commit来触发mutations来修改数据。
    3. mutations接收到commit的请求，就会自动通过Mutate来修改state（数据中心里面的数据状态）里面的数据。
    4. 最后由store触发每一个调用它的组件的更新
