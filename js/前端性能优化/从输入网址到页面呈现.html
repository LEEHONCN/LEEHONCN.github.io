      # 从输入网址到页面呈现

      这是一个经典的前端问题。通过分析这个问题，可以清晰地分析出哪些流程可以进行优化，来减少用户等待时间。

      # 处理流程

      网络方面：

      • 浏览器进行域名解析（将url中的域名部分拿出来）

      • 将解析出的域名进行域名解析（拿到服务器的IP地址以及端口）

        • 浏览器缓存
        • 系统缓存
        • 路由器缓存
        • DNS缓存
        • DNS查找

      • 传输层与服务器进行TCP连接（有IP和端口就可以进行TCP连接了）

      • 应用层进行HTTP连接（使用TCP）

      • 网络层IP协议查询MAC地址（查询设备）

      服务器方面：

      • 服务器收到处理请求

      • 服务器发送相应报文，浏览器得到响应，得到页面代码。

      这部分主要是跟服务器性能，以及代码的算法效率等问题有关。也可以减少第一字节时间。

      客户端部分：

      • 解析html标签并构建DOM树

      • 处理CSS标记并构建CSSOM数

      • Render树渲染阶段 将 DOM 与 CSSOM 合并成一个渲染树。

      • 布局阶段 根据渲染树来布局，以计算每个节点的几何信息。

      • 绘制阶段 将各个节点绘制到屏幕上。

      # 优化概述

      # 三个方面

      • 静态资源优化
      • 接口访问优化
      • 页面渲染速度优化

      在操控门槛上依次递增，优化效果上越发没有这么明显。
