      # 网络方面优化

      概述：

      # 域名解析：

      • DNS负载均衡(CDN) 通过将资源部署到世界各地，用户访问时按照就近原则从最近的服务器获取资源，来提高获取资源的速度，也可以并发获取。

      • DNS预查询

        <link rel="dns-prefetch" href="http://...">

      # 浏览器缓存：

      强缓存,协商缓存

      当客户端请求某个资源时，获取缓存的流程如下：

      • 先根据这个资源的一些 http header 判断它是否命中强缓存，如果命中，则直接从本地获取缓存资源，不会发请求到服务器；

      • 当强缓存没有命中时，客户端会发送请求到服务器，服务器通过另一些request header验证这个资源是否命中协商缓存，称为http再验证，如果命中，服务器将请求返回，但不返回资源，而是告诉客户端直接从缓存中获取，客户端收到返回后就会从缓存中获取资源；

      # HTTP连接：

      • 静态资源优化：
        • 将静态资源打包合并减少HTTP请求数
        • 将多小图片使用雪碧图合成一张图片，不仅HTTP请求数减少，并且总的字节数也减少。
        • 将图片以字节码字符串的方式存在于代码中。base64。
      • 懒加载
      • 使用防抖节流算法减少http请求数。
      • 增加并发数：使用HTTP2
      • 避免重定向？
      • cookie优化，cookie越多会导致请求头越大
      • 启用GZIP压缩（Accept-Encoding：g-zip）
      • 合理利用HTTP缓存，通过设置Expires
